1. 数据库关：SQL 连接/嵌套不是背模板，是先认关系

Day2 数据库的主战场是“信息分散在多张表”。你先看图，把表之间的桥想清楚，再写 SQL。

DB-1 普通 JOIN：姓名、课程名、成绩同时出现

题干：查询选修了课程且成绩不低于 90 分的学生姓名、课程名和成绩。

先写题眼
输出列分散在 Student、Course、SC 三张表；SC 是桥；需要普通 JOIN，因为只要有选课记录的学生。
展开标准答案与解析

标准 SQL：

SELECT S.Sname, C.Cname, SC.Grade FROM Student AS S JOIN SC ON S.Sno = SC.Sno JOIN Course AS C ON SC.Cno = C.Cno WHERE SC.Grade >= 90;

为什么这样写：学生姓名在 Student，课程名在 Course，成绩在 SC，三张表必须通过 Sno 和 Cno 接起来。成绩不低于 90 是单行条件，所以放 WHERE，不放 HAVING。

常见错因：只写 Student JOIN Course 会缺桥；把 Grade 条件放 HAVING 是把普通行条件误当分组条件；SELECT 里写 Cname 但 FROM 没连 Course 会直接失去来源。

变式提醒：如果题干改成“即使没选课也显示学生”，主体不能丢，就改 LEFT JOIN。

DB-2 LEFT JOIN：没有记录也要保留主体

题干：统计每个学生的选课门数，要求没有选课的学生也显示，门数为 0。

展开标准答案与解析

标准 SQL：

SELECT S.Sno, S.Sname, COUNT(SC.Cno) AS course_count FROM Student AS S LEFT JOIN SC ON S.Sno = SC.Sno GROUP BY S.Sno, S.Sname;

题眼：“没有选课的学生也显示”是 LEFT JOIN 的强信号。左表 Student 是主体，即使右表 SC 没匹配也保留。

为什么 COUNT(SC.Cno)：没有选课时，右表字段是 NULL，COUNT(SC.Cno) 不数 NULL，所以会得到 0。若写 COUNT(*)，左表保留下来的那一行也会被数成 1，可能错。

变式提醒：“没有被任何学生选的课程也显示”时，主体变成 Course，要从 Course LEFT JOIN SC。

DB-3 自连接：一张 emp 表扮演两个角色

题干：emp(num, name, dept, salary, boss)。查询每位员工姓名及其老板姓名。

展开标准答案与解析

标准 SQL：

SELECT E.name AS employee_name, B.name AS boss_name FROM emp AS E JOIN emp AS B ON E.boss = B.num;

题眼：员工和老板都在 emp 表里。一张表要当两个人用，就必须起两个别名：E 表示员工，B 表示老板。

为什么不是两张真实表：题目只有一个 emp 表，boss 字段存的是老板的 num。自连接的本质是“同一张表复制两份角色视图”。

变式提醒：如果题干要求“没有老板的员工也显示”，用 LEFT JOIN；如果问“不是任何人的 boss”，不是这条自连接，而是排除出现在 boss 列中的人。

DB-4 全部/所有：双重否定才是稳模板

题干：查询选修了全部课程的学生学号和姓名。

展开标准答案与解析

标准 SQL：

SELECT S.Sno, S.Sname FROM Student AS S WHERE NOT EXISTS ( SELECT * FROM Course AS C WHERE NOT EXISTS ( SELECT * FROM SC WHERE SC.Sno = S.Sno AND SC.Cno = C.Cno ) );

人话翻译：选修全部课程 = 不存在一门课程 C，使得这个学生 S 没有选 C。外层 NOT EXISTS 否定“漏选的课程存在”，内层 NOT EXISTS 判断“这门课没有对应选课记录”。

为什么难：“全部”不是简单 COUNT，也不是随手 JOIN。COUNT 写法也可以，但更容易被重复记录和 DISTINCT 坑到；双重 NOT EXISTS 是关系除法风味题的稳模板。

变式提醒：“选修了张老师教的全部课程”时，Course 子查询先筛张老师的课程，再做同样双重否定。

2. AI 关：参考习题名词要变成可调用公式和场景

AI Day2 容易出现一堆新词：LDA、熵、SVM、AdaBoost、正则化。闯关时不要背孤立定义，要回答“它解决什么问题，公式里的量是什么意思，题干怎么识别”。

决策树公式读法
H(D) = -∑k pklog2pk
熵 H(D) 衡量一堆样本有多混乱；越纯越接近 0。
Gain(D,A) = H(D) - ∑v |Dv||D|H(Dv)
信息增益 = 划分前混乱度 - 划分后加权混乱度。
SVM / AdaBoost 题眼
maximize margin ⇔ minimize 12||w||2
SVM 找最大间隔分类面，离分界面最近的点叫支持向量。
α = 12ln1-εε
AdaBoost 中错误率越小，弱分类器权重 alpha 越大；错分样本下一轮权重会上升。

AI-1 决策树信息增益：哪个属性先问？

题干：共有 8 个训练样本，正负各 4 个，所以整体熵 H(D)=1。属性 A 划分成两组：A=0 时 3 正 1 负，A=1 时 1 正 3 负；属性 B 划分成两组：B=0 时 4 正 0 负，B=1 时 0 正 4 负。问根节点应优先选哪个属性？

展开标准答案与解析

答案：选属性 B。

理由：整体正负各半，熵是 1。属性 B 划分后，每个子集都只有一种类别，子集熵都是 0，加权后仍是 0，所以 Gain(D,B)=1-0=1。

A 为什么不如 B：A 的两个子集分别是 3:1 和 1:3，都还混杂，熵不为 0，划分后仍有不确定性，因此信息增益小于 B。

考试迁移：你不一定要精确背每个小数。看到一个属性能把样本分成纯组，另一个还混，根节点优先选能让熵下降更多的那个。

AI-2 SVM：为什么不是“离所有点最近”的线？

题干：简答：SVM 为什么要最大化分类间隔？支持向量在其中起什么作用？

展开标准答案与解析

标准答法：SVM 在能正确分类训练样本的前提下，选择离两类样本边界最近点都尽可能远的分类超平面，即最大化分类间隔。支持向量是离分类超平面最近、真正决定间隔大小的训练样本。最大间隔让分类边界对小扰动更不敏感，通常有更好的泛化能力。

为什么不是“离所有点最近”：分类边界如果离样本太近，一点点噪声就可能把样本推到另一侧。SVM 要的是离最近的关键点也尽量远，不是贴着点走。

题眼：最大间隔、支持向量、超平面、泛化。如果出现核函数，再补一句“核函数用于把非线性问题映射到高维空间处理”。

AI-3 AdaBoost：错分样本为什么权重变大？

题干：某轮弱分类器错误率 ε=0.25。下一轮训练时，错分样本权重会变大还是变小？这个弱分类器的权重 α 是正还是负？

展开标准答案与解析

答案：错分样本权重变大；弱分类器权重 α 为正。

α = 12ln1-0.250.25 = 12ln3 > 0

从零理解：AdaBoost 像一个不断纠错的老师。上一轮做错的样本，下一轮要提高权重，让新弱分类器更认真处理这些难点。错误率 0.25 小于 0.5，说明这个弱分类器比乱猜强，所以它在最终集成中有正贡献。

常见坑：不是所有弱分类器都有正权重。若错误率超过 0.5，说明比乱猜还差，公式会给出负的 alpha，实际训练通常会处理或丢弃这种分类器。

AI-4 L2 正则化：为什么 lambda 不能乱取？

题干：简答：L2 正则项为什么能缓解过拟合？lambda 越大越好吗？lambda 能取负数吗？

展开标准答案与解析

标准答法：L2 正则化在原损失后加入权重平方和惩罚，使过大的参数付出代价，从而限制模型复杂度，降低模型为了拟合训练噪声而把参数调得很夸张的风险，因此可缓解过拟合。

lambda 不是越大越好：lambda 太小，约束不够，仍可能过拟合；lambda 太大，模型被限制得过死，可能欠拟合，训练集和测试集都表现不好。

lambda 不能取负：负数会奖励参数变大，和“惩罚复杂模型”的目标相反，可能让优化不稳定。

3. 软工关：同一题干，不同图回答不同问题

软工 Day2 的重点不是“我见过这个图”，而是题干换一个问法，你知道该画哪种图。下面用同一个实验预约系统场景训练。

SE-1 题干分流：这题到底画什么图？

题干：学生可以查询可预约实验、提交实验预约申请、取消自己的预约；教师可以审核预约申请；管理员可以维护实验室和实验项。若学生提交申请时实验室时间冲突，系统提示冲突并拒绝提交。

问法 A：“请画出系统主要功能及参与者关系。”

问法 B：“请画出学生提交预约申请时系统内部对象之间的消息交互。”

问法 C：“请画出预约记录可能经历的状态变化。”

展开标准答案与解析

A：用例图。题眼是“主要功能及参与者关系”。参与者包括学生、教师、管理员；用例包括查询可预约实验、提交预约申请、取消预约、审核预约、维护实验室、维护实验项。

B：顺序图。题眼是“提交预约申请时对象之间的消息交互”。生命线可以有学生界面、预约控制器、实验室日程/冲突检查服务、预约记录、通知服务。消息大致是提交申请 -> 检查冲突 -> 创建预约或返回冲突提示 -> 通知。

C：状态图。题眼是“预约记录的状态变化”。状态可包括草稿/已提交、待审核、已通过、已拒绝、已取消、冲突拒绝等，触发事件是提交、审核通过、审核拒绝、取消、检测冲突。

常见错因：把“提交预约申请”画成用例图流程细节；把“状态变化”画成活动图；类图里把“查询、提交、取消”这种动作当类。

SE-2 类图：名词不是全都能变成类

题干：实验预约系统保存学生、教师、实验室、实验项、预约记录。每条预约记录关联一个学生、一个实验项和一个时间段，并记录状态。教师审核预约记录。

展开标准答案与解析

候选类：Student、Teacher、LabRoom、ExperimentItem、Reservation。

Reservation 的关键属性：startTime、endTime、status、createdAt。状态不一定单独成类，除非题目要求复杂状态对象；一般作为属性即可。

关键关联：一个 Student 可有多条 Reservation；一条 Reservation 属于一个 Student；一个 ExperimentItem 可被多条 Reservation 预约；一条 Reservation 对应一个 ExperimentItem；Teacher 审核 Reservation；ExperimentItem 可能安排在 LabRoom。

为什么“查询、提交、取消”不是类：它们是动作，通常变成方法、用例或消息，不是系统长期保存的对象。

自查表：类图必须检查类名、属性、方法、关联、多重性。考试如果时间紧，至少把类和关联画对，多重性写核心的 1、0..*。

4. 离散关：证明题先写第一句，别直接硬推

Day2 离散只做证明入口。你现在要练的不是天才式推导，而是看到目标就知道第一句话。

目标第一句话为什么
证明 ∀x P(x)任取一个对象 a。全称要证明所有对象，所以先拿一个任意对象。
证明 ∃x P(x)构造/取一个具体对象 a。存在只需要给出一个见证。
证明 P → Q假设 P 成立。蕴涵证明通常从前件出发推出后件。
证明 Γ ⊨ Q设解释 I 使 Γ 中公式全真。语义推论要证明前提全真时结论必真。
证明不成立给一个反例解释/赋值。只要前提真、结论假，就能打破语义蕴含。

D-1 语义蕴含：{P -> Q, P} ⊨ Q

展开标准答案与解析

标准证明：设任意赋值 v 使 P -> Q 与 P 都为真。因为 P 为真，且 P -> Q 为真，若 Q 为假则 P -> Q 会为假，矛盾。因此 Q 必为真。所以 {P -> Q, P} ⊨ Q。

题眼：这是语义版的假言推理。证明语义蕴含不要一上来写“显然”，要按“前提全真 -> 结论必真”写。

变式提醒：{P -> Q, Q} ⊨ P 不成立，这是肯定后件谬误，可以取 P 假、Q 真作为反例。

D-2 反例：P ∨ Q ⊨ P 是否成立？

展开标准答案与解析

不成立。取 P 为假、Q 为真，则 P ∨ Q 为真，但 P 为假。这就是前提真、结论假的反例，所以 P ∨ Q ⊭ P。

为什么这个反例够用：语义蕴含要求所有赋值下都不能出现前提真结论假。只要找到一个反例，蕴含就失败。

常见错因：看到 P ∨ Q 里有 P，就以为能推出 P。析取为真可能是 Q 支撑的，不能单独推出 P。

D-3 谓词翻译：所有预约都属于某个学生

符号约定：R(x) 表示 x 是预约记录；S(y) 表示 y 是学生；Belong(x,y) 表示预约 x 属于学生 y。

展开标准答案与解析
∀x(R(x) → ∃y(S(y) ∧ Belong(x,y)))

读法：任意一个 x，如果 x 是预约记录，那么存在一个 y，使得 y 是学生并且 x 属于 y。

为什么外层用蕴涵：“所有预约都……”只约束预约对象。对不是预约的 x，不需要它满足后半句，所以写 R(x) -> ...。

为什么内层用合取：存在的是一个真正的学生 y，而且 Belong(x,y) 成立，所以是 S(y) and Belong(x,y)。
