GitHub Pages

发布前 5 步

流程越固定，越不容易把本地临时文件、断链或隐藏入口发出去。

1

跑静态检查

在项目根目录运行 npm run validate，先让脚本检查链接、脚本语法、课程入口和私有路径。

2

手动打开主线

依次打开 index.html、start_here.html、practice_lab.html、dashboard.html、resources.html 和 progress_vault.html。

3

抽查全科资料

至少点开数据库、人工智能、软件工程、军事理论、离散数学、马原和操作系统的必做资源。

4

提交仓库根目录

保留 .nojekyll、package.json、README.md、docs/、data/ 和核心 HTML 文件。

5

开启 Pages

GitHub Pages 的 Source 选择 GitHub Actions；推送后 Actions 会先执行静态检查，再发布站点。

✓

展示给同学

对外只推荐学生首页、课程岛、资源库和仪表盘；本页用于维护者复核，不占学习动线。

+

扩展新课程

维护者先用 course_pack_builder.html 生成课程与资源片段，再补 data/resource_quality.js 并按质量清单人工校对。

Q

校对内容质量

用 quality_board.html 检查 reviewed、exam-like、archive，记录待改进备注并回写稳定结论。

发布检查台 只记录项目能否顺利交付给学习者。正式学习请从 学生首页、继续学习仪表盘 或 精品资源库 进入。
