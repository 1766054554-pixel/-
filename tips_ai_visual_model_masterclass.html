0. 先有整体故事：AI 模型其实都在回答同一个问题

你现在看 AI 会烦，是因为每个 PPT 都像在丢新名字。但考试里，大多数模型题都可以回到同一个框架：

输入数据 x 进来，模型 f 产生预测 y_hat；损失函数 L 批改预测哪里错；训练算法调整参数，让下次错得少一点；最后拿新数据检验泛化能力。
把这张图记住，后面每个模型都只是“模型 f”的不同写法。
模型它把 f 写成什么样最适合处理什么考试高频问法
线性/逻辑回归一条线或一个超平面结构简单、需要公式计算损失函数、sigmoid、分类概率
决策树一串 if-else 问题规则解释、表格特征熵、信息增益、过拟合、剪枝
SVM最大间隔分界面中小规模分类、边界清楚支持向量、间隔、核函数
神经网络多层可调打分器复杂非线性规律激活函数、BP、过拟合
CNN小窗口在图像上滑动图片、局部空间结构卷积核、参数共享、池化
RNN/LSTM按顺序读，并带着记忆句子、时间序列隐藏状态、长期依赖、门控
Transformer每个 token 对全局做注意力长文本、多模态、生成理解Q/K/V、self-attention、并行

1. 新词先铺地基：每个词都要知道它在闭环里站哪

特征 feature输入 x 的每一项信息。房价模型里的面积、楼层；图片里的像素；文本里的 token 向量，都是特征。
标签 label标准答案 y。分类题标签是类别，回归题标签是数值。没有标签时通常进入无监督学习。
参数 parameter模型要学出来的数字。线性模型的 w、神经网络连接权重、卷积核里的数字，都属于参数。
损失 loss模型做错的程度。回归常用平方误差，分类常用交叉熵。它像批改卷子的分数。
训练 training用训练数据不断调参数，让损失下降。不是“模型自己突然懂了”，而是参数被一步步改了。
泛化 generalization模型对没见过的新题也做得好。训练集好但测试集差，就是过拟合。
考试答题句式：某模型接收输入特征 x，通过参数化函数得到预测 y_hat，再用损失函数衡量预测和真实标签 y 的差异，并通过训练调整参数。模型好不好，最终看泛化能力，而不是只看训练集表现。

2. 决策树：不是“树长得像树”，而是每个节点都在问一个最有用的问题

决策树适合从零理解，因为它最像人类做判断：先问一个问题，把样本分成几堆；再在每堆里继续问。关键不是树形图好看，而是“第一个问题该问谁”。

决策树图要看“问题如何把样本分纯”，不是只记树的形状。
Entropy(D) =− ∑k pk log2(pk)
  • p_k：当前数据集中第 k 类样本的比例。
  • 熵越大，类别越混；熵越小，越纯。
Gain(D,A) = Entropy(D) − ∑v |Dv||D| Entropy(Dv)
  • A：准备用来划分的特征。
  • D_v：按特征 A 的某个取值 v 分出来的子集。
  • 信息增益越大，说明这个特征越能把混乱样本分清。
决策树简答题怎么写丰满？

决策树通过一系列特征测试把样本递归划分到叶节点。每个内部节点选择一个划分特征，常用信息增益或信息增益率衡量划分前后不确定性下降程度。优点是可解释性强、能处理离散特征；缺点是树过深时容易过拟合，因此常用剪枝、限制深度、最小样本数等方法提升泛化能力。

3. 神经网络：先别想大模型，先看一个“会调权重的打分器”

神经网络的基本单元很像打分表：每个输入特征乘一个权重，加上偏置，再经过激活函数。很多单元叠起来，模型就能表达复杂的非线性边界。

考试问神经网络时，先把它还原成 z、激活、损失、反向传播四步。
z= w1x1 + w2x2 +⋯+ wdxd +b = w⊤x+b
  • w：权重，表示某个特征对结果影响多大。
  • b：偏置，相当于整体分数的起点。
  • z：激活前的原始打分。
ReLU(z) = max(0,z)
  • z 小于 0 就压成 0，大于 0 就保留。
  • 作用：引入非线性，让网络能表达复杂模式。
BP 反向传播别背成黑魔法：前向传播先算预测，损失函数给出“错多少”，反向传播用链式法则把责任从输出层分摊回前面每一层，最后更新权重。简答题写“链式法则 + 梯度 + 更新参数”就有骨架。

4. CNN：图片不是一串数字，图片有“附近关系”

如果把图片拉平成一长串数字，模型会忘记哪个像素和哪个像素挨着。CNN 的核心是局部连接和参数共享：同一个小窗口在整张图片上滑动，寻找边缘、纹理、角点等局部模式。

CNN 考试标准答案模板

CNN 适合图像任务，因为图像具有局部空间结构。卷积层通过可学习卷积核在局部区域滑动，提取边缘、纹理等局部特征，并通过参数共享减少参数量；池化层对局部响应进行下采样，降低特征图尺寸和计算量，并提高一定的平移鲁棒性；ReLU 等激活函数引入非线性，使网络能表示复杂图像模式。

5. RNN/LSTM：序列题的关键词是“顺序”和“记忆”

句子、语音、股票、传感器数据都有时间顺序。RNN 的思想是每一步读当前输入 x_t，同时带着上一时刻隐藏状态 h_{t-1}。问题是路径太长时，早期信息会在训练中变弱，于是 LSTM 加了门控。

ht = activation ( Wxxt + Whht−1 +b )
  • x_t：当前时刻输入，比如当前词。
  • h_{t-1}：上一时刻记忆。
  • h_t：读完当前输入后的新记忆。
LSTM 为什么缓解长期依赖？

普通 RNN 在长序列中需要把梯度跨很多时间步传回前面，容易出现梯度消失或爆炸，导致早期信息难以学习。LSTM 引入记忆单元和门控机制：遗忘门控制旧信息保留多少，输入门控制新信息写入多少，输出门控制当前输出多少。这样重要信息可以沿较长时间传递，从而缓解长期依赖困难。

6. Transformer：不排队传话，而是每个 token 直接问全班“我该看谁”

RNN 是顺序读，远处信息要一步步传过来。Transformer 的 self-attention 让每个 token 和所有 token 直接计算相关性，所以能并行，也更擅长长距离依赖。

Attention(Q,K,V) = softmax ( QK⊤ √dk ) V
  • Q：查询，表示当前 token 想找什么。
  • K：键，表示每个 token 能被匹配的线索。
  • V：值，表示真正要被加权汇总的信息。
  • 除以 sqrt(d_k)：避免点积过大导致 softmax 太尖，训练不稳定。
BERT/GPT/ViT 怎么和 Transformer 连起来：BERT 偏理解，常说双向上下文和 Encoder-only；GPT 偏生成，常说自回归和 Decoder-only；ViT 把图片切成 patch 当 token，再用 Transformer 处理视觉任务。别把它们当三个孤立名词。

题 2：神经网络简答题

为什么多层神经网络中需要非线性激活函数？

看标准解析

标准答法：如果网络每层都只是线性变换，那么多层线性变换复合后仍然等价于一层线性变换，表达能力不会本质增强。非线性激活函数让模型能够表示复杂的非线性关系，例如 ReLU 将负值压为 0、正值保留，使网络可以学习更复杂的分类边界或函数映射。

题眼：“为什么需要激活函数”不是问 ReLU 公式本身，而是问非线性表达能力。

题 3：模型辨析题

“某模型把输入句子按词逐个处理，用 h_t 保存到当前位置的上下文；但长句中早期信息容易衰减，于是引入门控记忆单元。”这描述的是哪组模型？为什么？

看标准解析

答案：这是 RNN/LSTM。RNN 按时间步处理序列，用隐藏状态 h_t 保存当前上下文；普通 RNN 在长序列中容易出现长期依赖困难，LSTM 通过记忆单元和遗忘门、输入门、输出门控制信息流，缓解早期信息难以保留的问题。

为什么不是 CNN/Transformer：CNN 关键词是卷积核和局部窗口；Transformer 关键词是 self-attention、Q/K/V 和并行全局依赖。题干明确说逐个处理和隐藏状态，先切 RNN。

题 4：Transformer 公式解释

解释下面注意力公式中 Q、K、V 分别代表什么，以及为什么 Transformer 比 RNN 更易并行。

Attention(Q,K,V) = softmax( QK⊤ √dk )V
看标准解析

标准答法：Q 表示查询，表示当前 token 想从其他 token 获取什么信息；K 表示键，用于和 Q 计算相关性；V 表示值，是最终被加权汇总的信息。QK^T 计算 token 间相关性，softmax 将相关性变成权重，再对 V 加权求和。Transformer 不需要像 RNN 那样按时间步串行传递隐藏状态，各位置的 attention 可以并行计算，因此更易并行并能直接建模长距离依赖。

今天的过关线：你不需要把所有公式数值算熟，但要能做到：看到图像想到 CNN，看到序列隐藏状态想到 RNN/LSTM，看到 Q/K/V 想到 Transformer，看到熵/增益想到决策树；并且每个模型都能说出“为什么它要这样设计”。
