2. SQL 速记：按数据库真正执行顺序想

FROM
WHERE
GROUP BY
HAVING
SELECT
ORDER BY
JOIN 题目跨表就找桥：学生-选课-课程，员工-部门，订单-商品。桥通常是主键/外键。
LEFT JOIN 出现“没有”“即使没有也显示”“0 个也要统计”，优先想到左连接和 COALESCE。
GROUP BY/HAVING “每个部门”“平均工资”“人数超过 2”先分组。WHERE 筛单行，HAVING 筛分组。
全部/所有 能计数就写 COUNT(DISTINCT x) = (SELECT COUNT(*) FROM 全集表)。不能计数时用双重 NOT EXISTS。 
每组最高模板：
WITH mx AS (
  SELECT dept_id, MAX(salary) AS max_salary
  FROM employee
  GROUP BY dept_id
)
SELECT e.*
FROM employee e
JOIN mx ON e.dept_id = mx.dept_id AND e.salary = mx.max_salary;

3. 关系代数速记：中文查询翻译成符号

符号人话题干信号写法提醒
σ筛行工资大于、姓名为、课程名是条件写在下标。
Π取列查询姓名、输出学号通常放最外层。
×笛卡尔积FROM R,S 的原始两两组合常和 σ 搭配表达连接条件。
⋈连接表跨表查询自然连接看同名属性；必要时先更名。
∪并集合并两个同类结果两边属性结构要能对应。
−差集没有、不包含、不是先写全集，再减去不合格集合。
ρ更名自己和自己比、两两比较自连接/找两家以上时常用。
÷除法所有、全部、每一个被除数保留“对象-全集项”二元关系。
常用结构： Π_输出列 ( σ_条件 ( 表1 ⋈ 表2 ⋈ 表3 ) ) SQL 互转骨架： SELECT A,B,D FROM R,S WHERE R.C=S.C => Π_A,B,D ( σ_R.C=S.C ( R × S ) ) 所有课程： Π_学号,课程号(选课) ÷ Π_课程号(课程) 没有选课： Π_学号(学生) − Π_学号(选课)

4. 后半大题速记：Day3 内容压缩版

候选键找右部没出现的属性 -> 求闭包 -> 能推出全部是超码 -> 再查最小性。
BCNF每条非平凡 X->Y，X 必须是超码；违反就按 X∪Y 和 R-(Y-X) 分解。
冲突图不同事务、同一数据项、至少一个写；先发生的事务指向后发生的事务。
2PL找第一次 unlock；之后若还有 lock 就违反。严格 2PL 看写锁是否持有到 commit/abort。
恢复未提交 Undo，用旧值；已提交 Redo 或确认，用新值。
ER实体、属性、联系、基数；M:N 或带属性的联系要转关系表。

题 1：SQL 分组筛选

06:00

employee(emp_id, name, dept_id, salary)，department(dept_id, dept_name)。查询平均工资超过 8000 的部门名和平均工资。

评分点和解析
2 分题型是 SQL 分组聚合。FROM employee JOIN department，连接条件是 dept_id。
2 分按部门分组，SELECT 里普通列 dept_name 应放入 GROUP BY 或使用 dept_id, dept_name 一起分组。
2 分平均工资超过 8000 是分组后的条件，必须写 HAVING AVG(salary) > 8000。
SELECT d.dept_name, AVG(e.salary) AS avg_salary
FROM employee e
JOIN department d ON e.dept_id = d.dept_id
GROUP BY d.dept_id, d.dept_name
HAVING AVG(e.salary) > 8000;

题 2：SQL 全部课程

07:00

student(sid, name)，course(cid, cname)，sc(sid, cid)。查询选修了所有课程的学生姓名。

评分点和解析
2 分题眼是“所有”，SQL 可以用 COUNT(DISTINCT sc.cid) 与课程总数比较。
2 分按学生分组，HAVING 中比较该学生已选课程数和 course 表课程总数。
2 分用 DISTINCT 防止重复选课记录干扰计数。
SELECT s.name
FROM student s
JOIN sc ON s.sid = sc.sid
GROUP BY s.sid, s.name
HAVING COUNT(DISTINCT sc.cid) = (SELECT COUNT(*) FROM course);

题 3：关系代数跨表查询

06:00

用关系代数表示：查询选修了“数据库”课程的学生姓名。关系：Student(sid, name)，SC(sid, cid)，Course(cid, cname)。

评分点和解析
2 分输出是学生姓名，最外层是 Π_name。
2 分需要 Student、SC、Course 三表连接。
2 分课程名条件用 σ_cname='数据库'。
Π_name ( σ_cname='数据库' ( Student ⋈ SC ⋈ Course ) )

如果属性重名或自然连接条件不清，考试可写带条件连接或先更名，表达清楚即可。

题 4：关系代数“没有”

06:00

用关系代数表示：查询没有选修任何课程的学生学号。

评分点和解析
2 分全集是所有学生学号 Π_sid(Student)。
2 分已选课学生是 Π_sid(SC)。
2 分没有选课 = 全体学生 - 已选课学生。
Π_sid(Student) − Π_sid(SC)

题 5：每组最高回表

08:00

employee(emp_id, name, dept_id, salary)。查询每个部门工资最高的员工姓名、部门号和工资，若并列最高都输出。

评分点和解析
2 分题眼是“每个部门最高”，不能 SELECT name, MAX(salary) 硬凑。
3 分先按 dept_id 求 MAX(salary)，再回原表匹配部门号和工资。
2 分用 JOIN 回表可保留并列最高员工。
WITH mx AS (
  SELECT dept_id, MAX(salary) AS max_salary
  FROM employee
  GROUP BY dept_id
)
SELECT e.name, e.dept_id, e.salary
FROM employee e
JOIN mx ON e.dept_id = mx.dept_id AND e.salary = mx.max_salary;

题 6：候选键与 BCNF

08:00

设 R(A,B,C,D)，F={A->B, B->C, C->D}。求候选键，并判断是否满足 BCNF。

评分点和解析
3 分A+：A->B，B->C，C->D，所以 A+=ABCD，A 是候选键。
3 分B+ = BCD，不含 A，所以 B 不是超码；C+ = CD，不含 A/B，所以 C 不是超码。
2 分B->C、C->D 左边不是超码，违反 BCNF，因此 R 不满足 BCNF。

题 7：事务调度与恢复

07:00

调度：r1(A), w2(A), w1(B), r2(B)。写冲突边并判断是否冲突可串行化。

评分点和解析
3 分A 上 r1(A) 与 w2(A) 冲突，得到 T1->T2。
3 分B 上 w1(B) 与 r2(B) 冲突，得到 T1->T2。
2 分优先图只有 T1->T2，无环，所以冲突可串行化，等价串行顺序为 T1, T2。

题 8：ER 图转关系模式

08:00

医院系统：医生可接诊多个患者，患者可被多个医生接诊；每次接诊有时间、诊断结果。请给出实体、联系、基数和关系模式。

评分点和解析
2 分实体：医生、患者。接诊因为有时间、诊断结果，应该实体化为接诊记录。
2 分医生与患者通过接诊记录形成 M:N；一条接诊记录对应一个医生和一个患者。
3 分关系模式：医生(医生号, 姓名, ...)，患者(患者号, 姓名, ...)，接诊(接诊号, 医生号, 患者号, 时间, 诊断结果)。若无接诊号，可用 医生号+患者号+时间 作为组合主键。
