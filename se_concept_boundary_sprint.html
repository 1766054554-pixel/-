1. 十讲主线：软工不是背词，是把软件做出来

从零看软工，可以把它想成一条生产线。考试里的单选、多选、判断和简答，很多都在问：某个概念到底属于哪一段、解决什么问题、和相邻概念有什么区别。

立项/过程选开发路线，管过程、产品、人员、风险和计划。
需求分析把用户想法变成可开发、可验证的需求和模型。
概要设计决定系统大结构：架构、模块、组件、接口、包。
详细设计把一个功能细化成类、对象、方法、消息、状态。
实现编码、版本协作、前后端接口、调试和构建。
测试用黑盒/白盒等方法发现缺陷，验证软件满足需求。
考试动作：看到题先问“它在哪一段”。问用户想要什么，是需求；问系统由哪些大块组成，是概要设计；问一次操作怎么发生，是详细设计；问等价类/覆盖，是测试；问 Git/前后端，多半在实现与协作上下文里。

2. 七组高频概念边界

容易混的概念从零解释考试识别信号
瀑布 / 增量 / 螺旋 / 敏捷瀑布按阶段顺序推进；增量一部分一部分交付；螺旋每轮都强调风险分析；敏捷用短迭代和持续反馈应对变化。需求稳定、文档完整 -> 瀑布；先交核心再扩展 -> 增量；高风险/新技术 -> 螺旋；需求变化快/频繁反馈 -> 敏捷。
过程 / 产品 / 人员管理过程管理管怎么开发；产品管理管交付物和版本；人员管理管团队角色、分工与沟通。官方样题问项目管理对象，常见答案是过程、产品、人员，对象管理不是这一组。
用户需求 / 系统需求用户需求偏用户语言，说明用户希望系统做什么；系统需求偏开发语言，写成更精确、可实现、可验证的规格。“面向客户/业务目标” -> 用户需求；“面向开发者/SRS/精确约束” -> 系统需求。
功能性 / 非功能性需求功能性需求说系统提供什么功能；非功能性需求说功能应该以什么质量完成，如性能、安全、可靠性、易用性。“查询、注册、提交、生成报表” -> 功能；“2 秒响应、支持 500 人、安全加密” -> 非功能。
概要设计 / 详细设计概要设计看系统整体结构和模块分解；详细设计看类、方法、对象交互、状态变化等可实现细节。组件图、包图、架构分层 -> 概要；类图、顺序图、状态图、操作合约 -> 详细。
黑盒 / 白盒测试黑盒不看内部代码，按输入输出和需求设计；白盒看内部结构，按语句、判定、路径、分支设计。等价类、边界值 -> 黑盒；语句覆盖、判定覆盖、分支覆盖、基本路径 -> 白盒。
故障 / 错误 / 失效故障是软件制品里的缺陷；错误是运行时产生的不正确内部状态；失效是对用户表现为功能不满足。代码/模型/文档里有问题 -> 故障；程序状态错 -> 错误；外部行为错 -> 失效。

4. 简答题模板：别只写散词

软工简答题通常不是让你写诗，而是要 定义 + 区别 + 关系/例子。下面四题来自十讲主线和往年题稳定信号。

模板 1｜简述故障、错误、失效三者区别与关系

展开参考答案
定义故障是代码、模型、文档等软件制品中的不正确描述或实现；错误是程序运行时出现的不正确内部状态；失效是软件对外不能提供规定功能或行为。
关系故障在某些输入或环境下可能触发错误，错误传播到系统边界时可能表现为失效，但不是每个故障都会被触发。
例子把条件写反是故障；变量值因此错误是错误；用户看到错误结果是失效。

模板 2｜功能性需求和非功能性需求怎么区分

展开参考答案
定义功能性需求描述系统应提供的服务、功能和行为；非功能性需求描述这些功能的质量约束、运行约束和开发约束。
例子“学生能选课”是功能性需求；“选课页面 2 秒内响应”“系统支持 500 人同时访问”“密码必须加密存储”是非功能性需求。
答题坑不要把“重要功能”叫非功能；非功能不是不重要，而是不直接描述一个具体业务功能。

模板 3｜概要设计和详细设计的区别

展开参考答案
概要设计确定系统总体结构、架构风格、模块/组件划分、主要接口和数据流。常见产物包括架构图、包图、组件图、模块结构图。
详细设计把模块和用例细化到类、方法、对象交互、状态变化和操作合约。常见产物包括设计类图、顺序图、状态图、活动图、详细设计规格说明。
考试识别题干问“系统由哪些子系统/模块组成”偏概要；问“一次注册/支付/查询操作如何一步步发生”偏详细。

模板 4｜黑盒测试和白盒测试各适合什么

展开参考答案
黑盒不关心内部实现，从需求规格、输入域和输出行为设计测试，常用等价类划分、边界值分析、错误推测、因果图等。
白盒基于程序内部结构设计测试，关注语句、判定、分支、路径是否被执行，常用语句覆盖、判定覆盖、条件覆盖、基本路径覆盖等。
往年题信号用户名长度和字符组成 -> 等价类；电话号码格式 -> 等价类/边界；给代码或分支条件 -> 白盒分支/判定覆盖。
