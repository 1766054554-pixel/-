# 简历包装：星光学习岛

## 项目名称

星光学习岛 Starlight Study Island

## 简历一句话

设计并实现一个面向大学期末冲刺的全科交互学习平台，覆盖数据库、人工智能、软件工程、军事理论、离散数学、马原和操作系统 7 门课，整合课程入口、深学讲义、真题交互卷、背诵闪卡、错题回炉、本地存档和静态站质量检查，帮助低基础学生把分散资料转化为可执行复习路线。

## 技术栈

- HTML / CSS / JavaScript
- localStorage 本地状态管理
- 本地进度导入/导出
- 响应式静态网站
- SVG 插画与微动效
- Manifest 驱动的模块配置
- GitHub Pages 可部署
- GitHub Actions 自动部署
- PWA 应用外壳与离线兜底
- Node.js 静态站质量校验
- GitHub Actions 自动检查

## 可以写进简历的 bullet

- 从真实期末复习场景出发，设计“课程入口 -> 深学讲义 -> 真题训练 -> 错题回炉 -> 考前速查”的学习闭环，覆盖 7 门课的不同考试形态。
- 将首页课程卡、学习路线和功能分区抽象为 `data/starlight_manifest.js` 配置清单，支持后续按课程包扩展，不需要重写主页结构。
- 设计统一 `course.html` 课程岛中转页，根据课程清单自动生成单科考试形态、三步路线和核心资源，减少旧页面平铺造成的迷路感。
- 设计 `resources.html` 精品资源库，基于 `data/resource_catalog.js` 支持课程、类型、优先级、关键词筛选和 localStorage 完成状态记录，把 80 多个页面收束成可执行目录。
- 设计 `dashboard.html` 继续学习仪表盘，读取资源完成状态生成总进度、必做完成度、每科进度、下一步建议和今日学习包，增强回访学习体验。
- 整理数据库、人工智能、软件工程、离散数学和操作系统的交互练习入口，其中操作系统收录 2019-2020、2020-2021、2024-2025 三套往年题与考点闪卡。
- 为离散数学设计“先写证明骨架再看解析”的主观题训练，为马原和军理设计低压力背诵、定位和召回模块，兼顾理工题型与文科闭卷/开卷场景。
- 设计星光海岛知识冒险记小游戏，用故事化题目前置、拟人化解析、收集奖励和低压力反馈提升复习专注度。
- 实现纯静态多页面学习平台，使用 localStorage 记录闯关星光值、错题、真题完成状态、资源库完成状态、今日学习包偏好和本地学习存档，无需后端即可持续追踪学习进度。
- 建立内容质量规范和资料使用记录，要求每个模块说明来源、适用考试场景、题型训练方式和待校对风险，减少“只好看但不好学”的问题。
- 编写 `npm run validate` 静态站校验脚本，并接入 GitHub Actions，检查首页必备区块、课程清单、内部链接、脚本语法和公开仓库敏感路径。
- 补齐开源协作外壳，包括 Issue 模板、PR 模板、CHANGELOG 和维护者发布手册，让项目从个人复习工具升级为可 fork、可审核、可持续扩展的学习平台。
- 补齐 GitHub Pages 发布体验，包括自动部署 workflow、应用图标、manifest、分享预览、404/离线兜底和首页分享入口，让项目更接近可公开传播的完整网页应用。

## 面试讲法

这个项目不是简单做一个刷题网站，而是从“低基础学生考前资料很多但不知道怎么学”的真实问题出发。我先把复习拆成几个关键动作：找到课程入口、理解核心知识、识别考试题型、做交互真题、回炉错题、保存进度。然后把这些动作包装成更容易坚持的学习体验，比如 3 题一段的小冒险、星光记忆宝箱、离散证明链条、操作系统真题包和文科背诵闸门。

工程上我选择纯静态实现，因为它适合快速开源和 GitHub Pages 部署。状态用 localStorage 保存，用户不登录也能保留自己的进度。课程入口、课程岛、继续学习仪表盘、精品资源库和精品模块由数据清单驱动，既能保持首页清晰，也方便把不同课程的题库、讲义和速查包接入同一套产品壳。为了让项目更像可维护产品，我还补了静态检查脚本和 GitHub Actions，避免链接失效、脚本报错或私人路径泄漏。

## 展示顺序

1. 先打开首页，说明七门课入口、学习路线和本地进度。
2. 点开任意课程岛，展示单科考试形态、三步路线和核心资源。
3. 打开继续学习仪表盘，展示下一步建议、今日学习包、必做完成度和每科进度。
4. 打开精品资源库，展示筛选、搜索、优先级和完成打勾。
5. 打开星光海岛小游戏，展示故事化闯关和错题保存。
6. 打开操作系统真题包，展示多套往年题的交互练习入口。
7. 打开离散真题舱，展示主观题先写骨架再看解析。
8. 打开数据库、AI 或软工交互卷，展示多题型练习和解析。
9. 打开马原或军理背诵页，展示文科冲刺的召回训练。
10. 打开错题宝箱和星光存档室，展示跨模块回炉闭环。
11. 展示首页“分享给同学”、PWA 外壳、404/离线兜底和 GitHub Pages 自动部署，说明项目具备公开传播能力。
12. 展示 `npm run validate`、GitHub Actions、Issue/PR 模板和 CHANGELOG，说明项目可维护性。
