# 资料来源与校对说明

本项目当前发布版来自一次真实期末冲刺场景，内容经过整理后作为学习产品样例保留。

## 当前样例内容

- 数据库：SQL、关系代数、范式、事务、ER 作图、2023A 整卷交互和考前变式训练。
- 人工智能：老师总纲、参考习题交互卷、模型故事线、公式模型闭环和生成/强化学习收口。
- 软件工程：官方五类题型交互卷、CodeArts 画图、纸质开卷定位、概念边界和往年题变式。
- 军事理论：闭卷背诵舱、数字/定义/简答骨架召回闸门和文科低成本包。
- 离散数学：零基础讲义、证明链条、公式速查、近三年真题交互卷。
- 马原：老师最后重点整理、纸质开卷索引、材料题定位与第一段训练。
- 操作系统：三套往年题交互卷和考点闪卡。
- 通用学习体验：星光海岛小游戏、错题宝箱、本地进度看板。
- 平台能力：星光存档室支持本地学习记录导出和导入。

## 来源原则

- 课程材料、老师重点、教材和真题是内容设计的主要依据。
- 所有解析都应经过人工校对，避免只依赖自动抽取。
- 真题类页面应尽量保留题型、题眼和作答动作，但公开发布时要遵守课程资料和版权边界。

## 公开版处理

为了让仓库更适合开源，公开版不默认提交本地原始抽取日志、私人路径和临时构建脚本。保留在页面中的内容应尽量是已经整理过、对学习者直接有用的版本。

如果你要继续维护课程资料，请在本文件中记录：

```text
日期：
课程：
资料类型：
新增/修改模块：
人工校对状态：
待确认问题：
```

稳定的资源质量默认状态写在 `data/resource_quality.js`：每个资源都有校对等级、来源依据、下一步改进点和更新时间。也可以在 `quality_board.html` 逐条资源记录本地巡检备注；该看板使用 `starlight-quality-audit-v1` 保存本地记录，确认后再把稳定结论写回 `data/resource_quality.js` 或本文件。
