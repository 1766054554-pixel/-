# 内容质量巡检清单

这份清单用来防止星光学习岛变成“资料很多但学不会”的题库。每个新增页面、题目和解析，都先按这里检查，再决定是否放到发布版入口。

## 一条总原则

学生打开页面后，应该更知道自己下一步怎么做，而不是只觉得“东西好多”。内容要服务考试动作：识别题型、想起定义、写出第一步、完成计算、组织证明、定位开卷资料、复盘错题。

## 发布等级

| 等级 | 含义 | 是否能进首页 |
| --- | --- | --- |
| `draft` | 初稿，可能只是资料摘录或自动整理 | 否 |
| `reviewed` | 人工读过，表达清楚，解析完整 | 可以进普通模块 |
| `exam-like` | 贴近真题题型，有变式和误区说明 | 可以进主线入口 |
| `archive` | 旧版材料或不再推荐路线 | 否 |

## 讲义质量

一份讲义至少要满足：

- **从零开始**：出现新词时先解释它为什么需要、解决什么问题，再给正式定义。
- **有主线**：不要平铺罗列概念，要告诉学生这些概念之间怎么连接。
- **有考试语言**：每个重点都要写出“卷面会怎么问”和“第一步该做什么”。
- **有例子**：新公式、新证明套路、新模型对比后面，要立刻跟一个小例子。
- **有重点标亮**：真正要背、要迁移、容易错的句子要醒目，而不是整页平均用力。
- **不堆重复**：同一知识点如果讲义已经细讲，学习舱就负责提醒、练习和回炉，不照抄一遍。

## 题目质量

一道好题不只是“不傻”，还要能训练迁移能力：

- **题眼明确**：题干里有真实考试会出现的信号，不靠生硬问定义。
- **错误选项真实**：干扰项要像常见误区，不要一眼看出正确项最长、最完整。
- **问法多样**：同一考点至少准备一种变式，比如换表述、换条件、换证明目标。
- **难度分层**：基础题负责启动，标准题负责拿分，变式题负责防考场卡住。
- **来源可追踪**：真题、参考习题、课件例题、老师重点、原创变式要标清。

## 解析质量

解析必须让学生下次能自己做，不能只给一句“所以选 B”。

每道题建议包含：

1. **题眼**：看到哪个词、符号、结构，就该想到哪个知识点。
2. **第一步**：考场上先写什么、先算什么、先画什么。
3. **关键理由**：为什么这样做，不只是结论。
4. **错因排查**：其他选项为什么错，或这类题最容易错在哪里。
5. **迁移提醒**：老师换个问法时，哪个信号仍然不变。

## 主观题和证明题

主观题不能只放参考答案。它必须训练“我怎么想到这一步”。

- 先让学生写骨架，再展开标准答案。
- 证明题要拆成：目标、可用定义/定理、第一句、连接句、结尾句。
- 马原材料题要拆成：题眼、原理、材料扣合句、第一段模板、容易空喊口号的地方。
- 软工/建模题要拆成：题干对象、边界、关系、图元素、箭头或约束检查。

## 学习舱与深学讲义的分工

- 深学讲义负责：完整解释、系统连接、例题带学。
- 学习舱负责：今天做什么、快速回忆、题眼识别、错题回炉。
- 真题舱负责：保留考试题型节奏，做完再看解析。
- 小游戏负责：降低启动成本，不替代完整学习。

如果两个页面出现大段重复，就要删掉其中一个的解释，把它改成练习、索引或回炉入口。

## 发布前最小检查

每次发布前问自己：

- 这个入口是不是给学生用的，不是给维护者或简历评委看的？
- 学生点进去 30 秒内能不能知道自己该干什么？
- 题目和解析是否贴近考试形式？
- 有没有新词没解释、公式没翻译、符号没铺垫？
- 有没有正确答案明显最长、错误选项太敷衍的问题？
- localStorage 记录是否能被错题宝箱或进度看板复用？

只有这些都过了，内容才算可以放进公开学习主线。
