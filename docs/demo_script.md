# Demo Script

这份脚本用于录屏、答辩、作品集展示或面试讲述。目标是在 3-5 分钟内说明：星光学习岛不是资料文件夹，而是一个可发布、可维护、可扩展的学习产品。

推荐从 `product_intro.html` 开场：先用沉浸式产品介绍页讲清“它不是资料合集，而是期末复习路径”，再进入 `index.html` 展示真实学习站。这样对外演示更像一个完整产品，而不是直接把访问者丢进资源库。

## 30 秒版本

星光学习岛是一个面向期末冲刺的纯静态交互学习平台。它把数据库、人工智能、软件工程、军事理论、离散数学、马原和操作系统的讲义、真题、闪卡、背诵和错题回炉统一成课程岛、真题训练台、精品资源库和继续学习仪表盘。学生不需要在资料堆里找路；维护者可以通过资源清单、质量元数据、课程包生成器和 GitHub Actions 继续扩展课程。

## 3 分钟展示路线

### 1. 产品介绍页：先讲清项目价值

打开 `product_intro.html`。

讲法：

- 这个页面是给访问者看的产品门面，不是学习主流程。
- 星图背景、层叠产品卡和互动演示视角分别讲课程岛、真题训练、错题回炉和开源扩展。
- 访问者先理解“为什么做、怎么用、怎么扩展”，再进入学习站。

展示动作：

1. 看首屏的产品定位和进入学习站按钮。
2. 滚到“沉浸展示”，指出四层产品卡。
3. 切换“低基础学生 / 考前刷题者 / 课程维护者 / 项目展示者”四个演示视角。

### 2. 首页：定位和第一入口

打开 `index.html`。

讲法：

- 这个平台服务的是低基础、时间紧、容易被资料堆吓住的期末复习者。
- 首页只保留三个主动作：选课程岛、查资料库、做真题训练。
- 七门课不是散文件，而是统一入口。
- “分享给同学”可以复制入口或调用系统分享，方便发布后传播。

展示动作：

1. 点“选择课程岛”滚到课程区。
2. 指出课程卡只保留考试形态、资源数量和入口。
3. 点“分享给同学”，展示复制或系统分享反馈。

### 3. 课程岛：单科从哪里学

打开任意课程，例如 `course.html?id=discrete` 或 `course.html?id=software-engineering`。

讲法：

- 课程岛负责把单科考试形态、三步路线和核心资源集中起来。
- 每门课都有本课筛选：主线资源、必做清单、真题卷、讲义/速查、回炉补洞。
- 完成状态写入同一个 localStorage key，资源库和仪表盘都能读取。

展示动作：

1. 切换“真题卷”和“讲义/速查”。
2. 标记一个资源完成。
3. 切到另一个课程岛，说明同一套结构可复用。

### 4. 精品资源库：全量资料不乱

打开 `resources.html`。

讲法：

- 平台现在有 80 多个资源，但学生默认不会看到混乱平铺。
- 资源库支持按课程、阶段、类型、优先级、质量等级和关键词筛选。
- 每个资源都有来源依据和下一步改进，方便公开后继续校对。

展示动作：

1. 搜索“离散 证明”或“OS 真题”。
2. 切换“只看真题级”。
3. 点“标记完成”，说明本地状态会同步到仪表盘。

### 5. 真题训练台：从看懂到会做

打开 `practice_lab.html`。

讲法：

- 真题训练台不让学生先看一堆资料，而是按动作进入：整卷校准、专项补洞、错题回炉。
- 数据库、离散、操作系统等都有交互真题或真题化训练入口。
- 离散主观题强调先写证明骨架再看解析，避免只背答案。

展示动作：

1. 选择一个课程或训练动作。
2. 打开一套交互卷。
3. 展示选择题即时反馈或主观题展开解析。

### 6. 仪表盘和存档：能继续学

打开 `dashboard.html` 和 `progress_vault.html`。

讲法：

- 仪表盘读取完成状态、向导路线和今日学习包，给出下一步建议。
- 存档室可以导出/导入本地学习状态，适合换设备和展示 demo。
- 纯静态站也能做到轻量状态管理，不需要后端。

展示动作：

1. 看必做完成度和课程进度。
2. 生成或查看今日学习包。
3. 打开存档室展示导出入口。

## 5 分钟技术讲述

如果面试官关心工程实现，可以按这个顺序讲：

1. 纯静态架构：HTML/CSS/JS + GitHub Pages，部署成本低。
2. 数据驱动入口：`data/starlight_manifest.js` 管课程和首页主线。
3. 资源目录：`data/resource_catalog.js` 把所有讲义、真题、闪卡和速查包结构化。
4. 质量层：`data/resource_quality.js` 标注 `draft`、`reviewed`、`exam-like`、`archive`，避免未校对内容进入主路径。
5. 本地状态：localStorage 保存完成状态、今日学习包、闯关、错题和存档。
6. 开源维护：Issue 模板、PR 模板、课程包生成器、质量看板、发布检查台。
7. 自动发布：GitHub Actions 先运行 `npm run validate`，再部署 Pages。
8. 产品外壳：manifest、icon、Open Graph、service worker、offline、404 和 robots。

## 项目亮点句

- 从真实期末救火场景抽象出可复用学习产品，而不是只做静态资料站。
- 用“课程岛 + 资源库 + 训练台 + 仪表盘”把复杂资料拆成清楚动作。
- 用质量元数据和静态校验约束内容质量，降低开源后继续维护的风险。
- 用本地状态和交互卷增强复习闭环，不登录也能保存进度。
- 用 GitHub Actions、Pages、PWA 外壳和发布兜底，把个人项目包装成可公开使用的平台。

## 展示时避免

- 不要说“这是我给自己临时做的资料堆”。
- 不要先展示维护页，学生主路径更重要。
- 不要从某个很长的讲义开始讲，先讲产品结构。
- 不要承诺所有题目都官方准确；要说它有质量层和来源校对机制，公开使用时以课程官方资料为准。
