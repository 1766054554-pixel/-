# 产品蓝图：星光学习岛

## 一句话定位

面向大学期末冲刺场景的治愈系学习陪跑平台，把“资料堆、真题卷、错题本”变成一条可执行、可反馈、可复盘的学习路线。

## 目标用户

- 考前时间紧、基础薄弱、不知道从哪里开始的学生。
- 看过讲义但不会做题，需要题眼识别和第一步训练的学生。
- 想把纸质开卷资料变成考场可定位工具的学生。
- 想通过轻游戏化降低分心和拖延的人。

## 核心体验闭环

1. **进入状态**：用星光海岛小游戏做 3 题一段的小冒险，降低开始阻力。
2. **建立路线**：新手上岛向导根据课程、当前状态和资源质量层生成三步路线，默认排除维护资料。
3. **继续学习**：仪表盘读取新手路线、资源完成状态和质量层，给出路线续航、总进度、必做完成度、每科进度和下一步建议。
4. **开始练题**：真题训练台按整卷校准、专项补洞、错题回炉组织练习入口，并优先推荐真题级和已校对资源。
5. **生成今日任务**：用户选择可用时间、科目和目标，系统自动从未完成资源里挑出今日学习包。
6. **筛选资源**：精品资源库按课程、类型、优先级和关键词收束入口，让用户先做最值得做的内容。
7. **深学与练习**：每个学科模块按“题眼 -> 第一动作 -> 标准表达 -> 变式提醒”组织。
8. **真题校准**：近三年真题做成互动卷，避免只看答案产生熟悉感。
9. **错题回炉**：错题宝箱汇总各页面本地记录，给出下一步补洞入口。

## 信息架构

- 首页：学生状态分流、三步复习路线、精选学习入口、本地进度。
- 新手上岛向导：`start_here.html` 根据课程和状态生成三步路线，解决第一次打开不知道点哪里的问题。
- 课程岛：`course.html` 根据课程清单、资源目录和质量元数据动态生成每门课的考试形态、推荐路线和本课主线资源。
- 真题训练台：`practice_lab.html` 根据资源库和质量元数据生成练题视图，支持科目筛选、训练模式筛选、完成打勾和 3 项训练包。
- 仪表盘：`dashboard.html` 根据新手路线、本地完成状态和质量元数据推荐下一批资源，展示每科推进情况，并按时间、科目、目标生成今日学习包。
- 资源库：`resources.html` 根据 `data/resource_catalog.js` 和 `data/resource_quality.js` 提供搜索、主线/真题级/全部/维护资料范围、课程、阶段、类型、优先级、校对等级筛选和完成打勾。
- 课程包生成器：`course_pack_builder.html` 面向维护者，把新课程表单转成课程清单和资源清单片段。
- 内容质量看板：`quality_board.html` 面向维护者，按资源记录校对等级、备注和待改进点。
- 游戏层：星光海岛知识冒险记。
- 学科层：数据库、人工智能、软件工程、军事理论、离散数学、马原、操作系统。
- 真题层：数据库 2023A、AI 参考习题综合卷、软工官方五类题型、离散近三年希冀卷、操作系统三套往年题。
- 回炉层：星光记忆宝箱。
- 仓库说明层：README、产品蓝图、课程包规范、简历讲述。它们面向项目维护和作品集展示，不进入学生学习首页。
- 开源协作层：Issue 模板、PR 模板、CHANGELOG、维护者发布手册和 GitHub Actions，服务后续同学继续维护。
- 配置层：`data/starlight_manifest.js` 管理首页主线和课程岛，`data/resource_catalog.js` 管理精品资源库，`data/resource_quality.js` 管理每个资源的默认校对等级、来源依据和下一步改进点。
- 存档层：`progress_vault.html` 支持本地学习记录导出、导入和清理，覆盖资源库完成状态、今日学习包偏好、闯关、真题、背诵和错题记录。

## 公开界面边界

公开网页只回答学生最关心的四个问题：

1. 我现在该从哪里开始？
2. 我应该学哪一块？
3. 我能不能做接近期末的真题？
4. 我错过的点下一次怎么补？

仓库文档可以讲开源路线、课程扩展、产品设计和简历亮点，但这些内容不放在 `index.html` 的学生入口里。首页的每一个卡片都必须指向真实学习动作，不能展示“给维护者看的说明”。

## 差异化亮点

- 不是传统题库：题目前有场景，解析有小动物口吻，错题叫记忆宝箱。
- 不是资料迷宫：新手上岛向导先给出三步路线，再把用户带到课程岛和训练台。
- 不是单纯美化：每个交互都服务学习动作，比如题眼识别、证明第一句、开卷定位。
- 不是一次性资料包：本地状态会持续记录，让“今天该做什么”有依据。
- 不是只记录进度：仪表盘会把新手路线和完成状态转化成下一步建议和今日学习包。
- 不是只给资料入口：真题训练台把资源转化成“现在开始练”的动作流。
- 不是乱堆旧页面：资源库把页面收束成精选入口，支持筛选和完成打勾。
- 不是只能个人维护：课程包生成器把扩展新课程变成结构化流程。
- 不是只靠口头说精品：资源质量元数据和内容质量看板把 reviewed、exam-like、来源依据和待改进点变成可追踪记录。
- 不是一次性展示项目：Issue 模板、PR 模板、CHANGELOG 和维护者发布手册让它更像能被接手的开源学习产品。
- 不是单设备玩具：学习记录可以导出成存档，在不同浏览器之间迁移。
- 可以开源复用：纯静态部署，未来可把课程配置抽成数据文件。

## 下一阶段产品化任务

1. 把旧页面中最有价值的模块统一成同一套设计组件。
2. 将课程、题库、奖励、路线继续抽成 `data/*.json` 或课程包脚本。
3. 让课程包生成器支持导入现有 JSON、批量资源行和字段校验。
4. 增加题目质量审核状态：来源、考点、难度、题型、解析完整度。
5. 增加“导入课程包”能力，让学弟学妹可以维护新一年的资料。
6. 增加移动端专注版：首页优先显示“继续上次任务”和“错题回炉”。

## 可以展示给面试官的工程点

- 产品设计：从真实学习痛点拆解用户路径。
- 前端实现：纯静态多页面应用、响应式布局、本地状态管理。
- 内容工程：把 PDF、PPT、真题和课堂重点转成结构化学习模块。
- 交互设计：故事化、收集系统、低压力反馈、错题闭环。
- 可维护性：发布版和素材库分离，后续课程可配置化。
