1. 老卷给出的 8 个警报

我这轮重新抽了 2014A、14级例卷、13级回忆、09年试题、12级片段和 2016 PDF。2016 的文字层仍然很差，只能确认文件和标题，不能假装完整读透；但 2014/13/09/12 已经足够告诉我们：北航数据库旧卷喜欢用客观题和小大题考“边界概念”。

旧卷信号你要形成的第一反应怕什么
SQL 语法选择题WHERE 不能写聚合条件，HAVING 才筛组；GROUP BY 后 SELECT 只能写分组列或聚合。凭语感选，而不是按 SQL 逻辑顺序验。
LIKE 匹配字面量 `%`遇到“倒数第几位是百分号”，立刻想 ESCAPE。把 `%` 当普通字符，或忘记“至少 4 位”需要一个必有前缀字符。
关系代数基本运算基本运算常见说法是 σ, Π, ×, ∪, −，连接/除/交/重命名可由基本运算派生或作为扩展使用。只认识中文，不认识卷面符号。
关系代数结果手算先看连接条件和同名列，再决定输出列；外连接要保留没匹配的行并补 NULL。把自然连接、笛卡尔积筛选、外连接混成一团。
二元关系/BCNF 客观题二元关系模式一定能达到 BCNF；但有部分依赖的多属性关系可能只到 1NF。看见“范式”就机械套 2NF/3NF，不先求键。
2PL 与可串行化2PL 推出冲突可串行化，但冲突可串行化不一定满足 2PL。把充分条件当必要条件。
索引/安全/完整性索引加速查找但不是安全机制，也不是每列都应该建；完整性约束保证正确性和相容性。把“数据库保护”泛化成所有优化工具。
ER 转关系计数实体先变表；M:N 联系一定另建表；1:1/1:N 联系通常可用外键合并。见联系就都建表，表数直接爆炸。
使用时机：做完 数据库临考变式训练营 后看本页；如果这里客观题错 4 道以上，不要急着刷整卷，回 Day1 关系代数、Day3 范式事务、Day6 往年题矩阵各补 10 分钟。

2. 关系代数符号：按卷面长相认，不按想象认

你之前指出得对：如果我只说“选择、投影”，但试卷写的是 σ 和 Π，那等于没有真正教到考试形态。这里把旧卷出现或容易出现的写法放一起。

σ条件(R)选择，筛行。对应 SQL 的 WHERE。例：σAge>18(Student)。
Π列(R) / ∏列(R)投影，筛列。旧卷可能用大写 Π，也可能抽取成 ∏。
R × S笛卡尔积。先暴力两两组合，再用 σ 筛连接条件。
R ⋈ S / R ▷◁ S自然连接。同名属性自动相等；如果没有同名属性，自然连接等价于笛卡尔积。
R ⟕ S / R ]▷◁ S左外连接。保留左表所有行，右边没匹配就补 NULL。
ρ新名(R)重命名。自连接、同一张表出现两次时特别有用，例如 I1, I2, I3。
R ∪ S, R − S, R ∩ S并、差、交。要求两边属性结构相容；“没有/不在”常用差集。
R ÷ S除法。看到“所有/全部/德国的所有邻国”要警觉；意思是找出能覆盖 S 中每个对象的左侧对象。
γ分组;聚合(R)扩展关系代数里的分组聚合。旧卷更常要求 SQL 写 GROUP BY/HAVING，但看到它不要慌。
旧卷里的典型互转：SELECT A,B,D FROM R,S WHERE R.C=S.C 对应 ΠA,B,D(σR.C=S.C(R×S))。顺序是 先 FROM 成 R×S，再 WHERE 成 σ，最后 SELECT 成 Π。

2.5 关系代数卷面闯关：先认长相，再写骨架

这块专门补“我在卷子上看到它到底该做什么”。你不需要把关系代数当数学怪物背，先把符号翻译成数据库动作：筛行、筛列、拼表、排除、覆盖全部。只要这五个动作稳住，SQL 和关系代数互转就不容易懵。

σC(R)先读作“在 R 里筛出满足 C 的行”。题干对应 WHERE。
ΠA,B(R)先读作“只留下 A、B 两列”。题干对应 SELECT。
σR.x=S.x(R×S)先读作“先暴力拼 R 和 S，再按相等条件筛”。题干对应 FROM R,S WHERE R.x=S.x。
R ⋈ S先读作“按同名属性自动拼表”。没有同名属性时退化成笛卡尔积。
R ÷ S先读作“找能覆盖 S 里每一个对象的 R 侧对象”。题干常写“所有、全部、每一门”。

闯关 1：把符号读成人话

ΠName(σCourseName='Database'(Student ⋈ Take ⋈ Course))

展开标准读法
先把 Student、Take、Course 通过学号和课程号连接起来，再筛出课程名为 Database 的记录，最后只投影学生姓名 Name。人话答案是：查询选修了 Database 课程的学生姓名。

闯关 2：把 SQL 改写成关系代数

SELECT S.name FROM Student S, Take T WHERE S.sid=T.sid AND T.cid='C01'

展开标准骨架
逻辑顺序是 FROM 到 WHERE 到 SELECT，所以可写成 ΠS.name(σS.sid=T.sid ∧ T.cid='C01'(Student S × Take T))。如果题目允许连接符号，也可把相等连接写成连接形式，但基础写法用 × + σ 最不容易丢步骤。

闯关 3：看到“全部”别硬套 JOIN

题干：查询选修了全部课程的学生学号。

展开标准起手
被除关系是 ΠSID,CID(Take)，表示每个学生选了哪些课；除数是 ΠCID(Course)，表示全部课程。关系代数核心骨架是 ΠSID,CID(Take) ÷ ΠCID(Course)。它的人话是：找一个 SID，使得 Course 里的每个 CID 都出现在这个学生的选课记录里。
临考 8 分钟用法：先遮住答案，把每道题读成人话；再写符号骨架；最后展开答案。你不是要在这里刷很多题，而是把“卷面符号 -> 数据库动作”这条神经接牢。

4. 主观题骨架：只练第一步和坑点

这里不再要求你写完整大题。临考前更重要的是：看到题干时，脑子里能自动弹出 “这题属于哪类、第一步写什么、哪句话能拿基础分”。

SQL：没有购买商品的顾客

旧卷信号：Customer、Goods、sale 三表，问“没有购买”。

展开参考骨架

首选 NOT EXISTS：外层枚举顾客，内层查 sale 中是否存在该顾客购买记录。不存在则留下。

SELECT C.cn
FROM Customer C
WHERE NOT EXISTS (
  SELECT 1
  FROM sale S
  WHERE S.cno = C.cno
);

扣分坑：不要把 Goods 强行连进来，因为题目只问“有没有购买记录”，sale 已经够判断。也不要写 NOT IN 后忘记 NULL 风险。

关系代数：与德国所有邻国都接壤

旧卷信号：“所有邻国都”就是除法或双重差集。

展开参考骨架

除数：德国的邻国集合 Π邻国(σ国名='德国'(邻国))。被除关系要整理成“候选国家, 德国邻国”两列，表示候选国家接壤了哪些国家。

如果不熟除法，就用人话记：找国家 X，使得 不存在一个德国邻国 Y，X 没有和 Y 接壤。这就是 SQL 里的双重 NOT EXISTS。

范式：R(A,B,C,D,E), F={AB→C, C→D, D→B, D→E}

旧卷信号：先求闭包和候选键，再谈 BCNF。

展开参考骨架

D+ = DBE，因为 D→B、D→E。AB+ = ABCDE，因为 AB→C，C→D，再 D→B/E。

候选键要覆盖 A，而 A 不在任何右部，所以候选键必须含 A；再试能推出 C/D/B/E 的搭配，旧卷答案给出 AB、AC、AD。BCNF 检查时看每条非平凡依赖左部是不是超键；C→D、D→B、D→E 左部不是超键，所以不满足 BCNF。

ER：5 个实体、2 个 1:1、3 个 M:N

旧卷信号：问“转换出来的关系表个数不可能是”。

展开参考骨架

5 个实体先至少 5 张表。3 个 M:N 联系一定各自变关系表，所以至少再加 3 张，得到 8 张。2 个 1:1 联系可以用外键合并进实体表，也可以单独建关系表，所以最多到 10 张。因此 7 张不可能。

扣分坑：M:N 一定另建表；1:1 不一定另建表。这句话很值钱。
