五类题型的第一动作

单选题

先找“不是/属于/最适合”。单选常考 概念边界：过程模型 vs OO模型，黑盒 vs 白盒，需求 vs 设计。

多选题

不要凭感觉全选。先在草稿上写同类集合，例如项目管理对象=过程/产品/人员；黑盒方法=等价类/边界值/错误推测/因果图/场景。

判断题

判断题常把“阶段、对象、方法”换位。看到绝对词如“只能、必须、一定、全部”，特别小心。

简答题

用三段式：定义一句 -> 关系一句 -> 例子/后果一句。例如故障、错误、失效就写传播链。

综合题

先标四层：需求分析、概要设计、详细设计、测试。材料题不是让你一口气画所有图，而是问哪层就交哪层产物。

图题

先写图的范围：系统边界、参与者/类/对象、关系、箭头、多重性。图旁必须有必要文字说明，别只丢一张图。

看综合题先做这张图里的分流。问法变了，答案层次就要变。

往年题迁移：三类材料题要会“换壳识别”

2020 实验上机安排系统

题眼：教师提交实验项目、安排机房、学生查询安排、管理员维护基础信息和新闻公告。常考 用例图、类图、顺序图。

迁移动作：角色先圈教师/学生/管理员/未登录用户；实体再圈课程实验项目、机房、安排、报表、新闻。

12级 选课管理系统

题眼：学生选课、教师登记成绩、教务管理员维护信息、开启/关闭课程注册。常考 用例图、事件流、等价类。

迁移动作：基本流写正常选课；扩展流写未开启、超3门、人数已满、人数不足取消。

14级 库存/仓库类系统

题眼：库存、入库、出库、状态变化、分支覆盖、电话号码/标识符等输入规则。常考 状态图、类图、测试用例。

迁移动作：状态图看对象生命周期；白盒题看判定分支；黑盒题按输入条件列等价类。

2020 实验排课题：类图、用例图、顺序图最小交卷骨架
用例图：系统边界写“实验上机安排系统”。参与者至少有教师、学生、管理员、未登录用户。教师用例：提交课程实验项目、安排实验机房、查询安排结果、打印报表。学生用例：查询课程实验项目、查询课程实验安排。管理员用例：管理教师/学生/课程实验、管理新闻公告、查询机房时间安排。所有登录用户有修改密码、查看新闻公告。
类图：优先抽实体类：User、Teacher、Student、Admin、Course、ExperimentProject、LabRoom、ExperimentSchedule、News、Report。关系别忘多重性：一个教师可提交多个实验项目，一个实验项目对应一个课程，一次安排关联一个实验项目和一个机房，一个管理员管理多条新闻。
顺序图：学生查询安排：Student -> System/Boundary -> ScheduleController -> ScheduleService -> ScheduleRepository -> ExperimentSchedule。管理员管理新闻：Admin -> NewsController -> NewsService -> NewsRepository -> News。顺序图重点是时间顺序和消息，不是把所有类都塞进去。
12级 选课题：事件流和等价类怎么写
课程注册基本流：学生登录 -> 进入课程注册 -> 系统显示课程目录 -> 学生选择课程 -> 系统检查人数/数量/条件 -> 保存选课记录 -> 提示成功。
课程注册扩展流：管理员未开启注册；学生选择超过3门；课程人数已满；课程人数低于5人被取消；开学两周后关闭注册。扩展流不是废话，它就是综合题的得分点。
用户名等价类：有效类=8-12位、英文字母和数字组合、不能全字母、不能全数字；无效类=少于8位、多于12位、全字母、全数字、含非英文字母/数字字符。

简答题模板：别背长段，背骨架

故障、错误、失效三者关系
展开参考答案

故障是软件内部存在的缺陷或问题根源，错误是程序执行时产生的错误状态，失效是软件外部表现出的功能不符合需求。三者通常形成传播链：故障被触发后导致错误状态，错误状态继续传播就可能表现为失效。例如数组越界缺陷是故障，运行时数据被破坏是错误，用户看到系统崩溃就是失效。

黑盒测试与白盒测试区别
展开参考答案

黑盒测试依据需求规格说明，不关心程序内部结构，重点检查输入输出和功能行为，常用等价类划分、边界值、错误推测、因果图和场景法。白盒测试依据程序内部结构和控制流，重点检查语句、判定、条件、路径等覆盖情况。考试看到“代码流程图、分支、路径”想到白盒；看到“输入规则、业务场景、规格说明”想到黑盒。

需求分析、概要设计、详细设计区别
展开参考答案

需求分析回答系统要做什么，关注用户目标、功能、非功能约束和领域概念，产出需求模型和需求规格。概要设计回答系统高层如何实现，关注架构、子系统、组件、接口和数据组织。详细设计回答模块内部如何实现，关注类职责、属性、方法、对象交互、状态转移和算法细节。

综合题开头第一段
展开参考答案

本题先按软件生命周期和面向对象建模流程分层处理：需求分析阶段识别参与者、用例、系统边界和领域概念；概要设计阶段将系统划分为组件/子系统并明确接口；详细设计阶段给出关键类、对象交互和状态变化；测试阶段根据需求和设计选择黑盒或白盒方法设计测试用例。
